// How it works

Three steps.
Under 60 seconds.

No setup, no account, no installation. Works in any browser.

1

Drop your file

Upload CSV, Excel, TSV, ODS or JSON. It loads directly into your browser using SQL.js — a real SQLite engine. Nothing is uploaded anywhere.

2

Ask in plain English

Type your question. Only the question — not your data — goes to Gemini AI to generate SQL. Your file stays on your device the entire time.

3

Get answer + insight

The SQL runs locally. Results appear in under a second with an AI insight — and optionally as a bar, line, pie or scatter chart.

🔒
Verify it yourself: Open DevTools → Network tab while running a query. You will not see your data leave your device.

// Builder, QueryAI
Age 15 Self-taught developer Built with SQL.js + Gemini API Open source on GitHub

I built QueryAI because I kept seeing people upload sensitive business data to ChatGPT — not because they wanted to, but because they had no better option. I wanted to build that better option.

Everything runs locally. I spent weeks on the SQL.js integration to make sure no row of your data ever touches a network request. Your data stays in your RAM, not on some server you don't control.

This is not a startup. There is no VC money, no growth team, no monetization plan for your data. It's a tool I built because it should exist — and because I wanted to prove that privacy and AI power don't have to be a tradeoff.

"Privacy shouldn't be a premium feature — it should be the default."
